# QR-Based Food Ordering and Restaurant Management System

A fully-functional, responsive, and robust restaurant food ordering web application built using Python **Django**, **MySQL**, **Bootstrap 5**, and the **Pillow/python-qrcode** libraries. This project is ideal for an engineering final-year project demonstration.

---

## Tech Stack

- **Backend**: Python Django (6.x)
- **Frontend**: HTML5, CSS3 (with custom premium styling, Outfit font), JavaScript, Bootstrap 5
- **Database**: MySQL (configured using `PyMySQL` driver wrapper)
- **QR Code Engine**: python-qrcode
- **Image Processing**: Pillow

---

## Project Modules

### 1. Customer Panel (Mobile-Friendly)
- **Table Auto-Detection**: Scanning a table's QR code routes to `/table/<number>/`, saving the table details inside the Django session (no sign-up required).
- **Interactive Menu**: Grouped categories, search bar, and clean dishes list with descriptions, images, and prices.
- **Cart Management**: Add items, increase/decrease quantities, remove items, and see real-time totals in a session-based cart.
- **Order Tracking**: After checking out, the customer is routed to a live order tracking page which uses **AJAX polling** to fetch status updates without refreshing.

### 2. Kitchen Panel (Staff Auth Required)
- **Incoming Queue**: Active orders (`Pending`, `Preparing`, `Ready`) sorted oldest-to-newest, showing table numbers, quantities, and elapsed time.
- **Status Updates**: Transitions order status (`Pending` ➔ `Preparing` ➔ `Ready` ➔ `Served`) in one click.
- **History log**: Lists the 15 most recently served orders.

### 3. Admin Panel (Manager Auth Required)
- **Dashboard Analytics**: Shows Total Orders count, Total Served Revenue (summed), and a table of the **Top 5 Best-Selling Dishes** (using Django aggregate annotations).
- **Category CRUD**: Manage food categories.
- **Menu CRUD**: Manage menu dishes including name, description, price, and image uploads.
- **Table CRUD**: Manage dining tables. Renaming or adding a table automatically generates/updates a high-quality QR code image stored in media.
- **Order Management**: Search orders by table number, filter by status, and view detailed receipts.

## Installation & Setup Instructions

### 1. Database Setup (MySQL)
Make sure your MySQL server is running (e.g. via XAMPP, WAMP, or standalone MySQL service).
1. Open your MySQL client (e.g., phpMyAdmin, MySQL Workbench, or Command Line).
2. Run the SQL statements inside `restaurant_db.sql` or run the following command to create the database:
   ```sql
   CREATE DATABASE restaurant_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

### 2. Dependencies Installation
Install the necessary python dependencies using pip:
```bash
pip install django pymysql pillow qrcode
```

### 3. Run Database Migrations
Generate and execute the Django migration files to create the database tables:
```bash
python manage.py makemigrations
python manage.py migrate
```

### 4. Populate Sample Data
Initialize the database with default staff accounts, dining tables, categories, and dishes (with colored dummy images generated by Pillow):
```bash
python populate_data.py
```

## Running the Application

Start the Django development server:
```bash
python manage.py runserver
```
Visit the following URLs:
- **Customer Panel (Digital Menu)**: [http://127.0.0.1:8000/](http://127.0.0.1:8000/)

## Running Automated Tests
The system includes automated unit tests for scanning routes, cart session calculations, checkout logic, and table QR code saving. Run the test suite:
```bash
python manage.py test
```
*(Tests are configured to fallback to an in-memory SQLite database connection automatically to run without needing MySQL server access).*
